# Introducing archflow v1.0.0

Today we're excited to announce the first stable release of **archflow** — an open-source Java framework for building, visualizing, and managing AI agent workflows.

<!-- truncate -->

## Why archflow?

The AI ecosystem is evolving rapidly, but Java developers have been underserved. While Python has LangChain and JavaScript has LangChain.js, Java lacked a comprehensive framework for orchestrating AI agents with visual tooling and enterprise features.

**archflow bridges this gap**, providing:

- A **visual workflow designer** that runs as a React component or standalone Web Component
- Deep **LangChain4j integration** with 15+ LLM providers out of the box
- **MCP (Model Context Protocol)** native support for standardized tool access
- Enterprise-grade **security, observability, and plugin architecture**

## Key Features

### Visual Workflow Designer

Design AI workflows visually with our React-based editor. Drag-and-drop nodes, configure connections, and deploy — no YAML or JSON editing required.

```jsx
import { ArchflowDesigner } from '@archflow/component';

<ArchflowDesigner
  workflowId="my-workflow"
  apiUrl="http://localhost:8080/api"
  onSave={(workflow) => console.log('Saved:', workflow)}
/>
```

### LangChain4j Native

archflow is built on LangChain4j 1.10.0, giving you access to OpenAI, Anthropic, Google, Mistral, Ollama, and 10+ more providers through a unified adapter pattern.

### Suspend & Resume Conversations

Workflows can pause to collect user input via dynamic forms, then resume exactly where they left off — perfect for multi-step customer support flows.

### Enterprise Ready

- **JWT + RBAC** with 4 built-in roles
- **OpenTelemetry** distributed tracing
- **Prometheus + Grafana** metrics dashboard
- **Audit logging** with database persistence
- **Two-level caching** (Caffeine + Redis) for LLM responses and embeddings

### Plugin Ecosystem

Extend archflow with custom tools, assistants, and agents. Discover and install extensions from the built-in marketplace.

```xml
  <artifactId>archflow-spring-boot-starter</artifactId>
  <version>1.0.0</version>

```
